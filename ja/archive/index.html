# デュアルブートシステム間で同じショートカットキーを使用する方法 ## 問題の背景 WindowsとLinuxなど、異なるOSをデュアルブートで利用している場合、それぞれのシステムでショートカットキーが異なることがあります。例えば: - スクリーンショットの撮影 - ウィンドウの切り替え - アプリケーションの起動 この差異により、OSを切り替えるたびに操作感が変わってしまい、生産性が低下する可能性があります。 ## 解決方法 ### 1. キーボードリマッピングツールの利用 [[0]]や[[1]]などのツールを使用して、キーボードレイアウトを統一することが可能です。 #### Windows側の設定: - [[2]]をインストール - カスタムキーマップを作成 - 設定をエクスポートしてバックアップ #### Linux側の設定: - `xmodmap`または[[3]]を使用 - Windowsと同じキーマップを適用 - 設定を`~/.Xmodmap`に保存 ### 2. 共通のショートカット設定 頻繁に使用する操作について、両OSで同じショートカットを割り当てます: | 操作 | 推奨ショートカット | |----------------|------------------| | スクリーンショット | [[4]] | | ウィンドウ切り替え | [[5]] | | ターミナル起動 | [[6]] | ### 3. 設定の同期 [[7]]や[[8]]を使用して、両OS間で設定ファイルを同期させる方法: 1. 共有パーティションまたはクラウドストレージを作成 2. 設定ファイルを共有場所に保存 3. 各OS起動時に自動的に設定を読み込むスクリプトを作成 ## 注意点 - 一部の組み合わせショートカットはOSによって競合する可能性があります - ハードウェア固有の機能キーは再マップできない場合があります - 変更前に現在のキーマップを必ずバックアップしてください ## 高度な設定 上級ユーザーは[[9]]を使用して、より細かいキーボード動作の制御が可能です。この方法では: 1. カーネルレベルでのキー処理を変更 2. より低レイテンシーの入力を実現 3. 特殊なハードウェアにも対応 ただし、システムの安定性に影響を与える可能性があるため、注意が必要です。 ## まとめ デュアルブート環境でも、適切なツールと設定により一貫したキーボード操作を実現できます。最初に時間をかけて設定を整えることで、長期的な生産性向上が見込めます。

# TP-LinkでDHCPを無効にした後にPPPoE接続する方法 TP-LinkルーターでDHCP機能を無効にした後でもPPPoE接続でインターネットを利用するには、以下の手順に従って設定を行ってください。 ## 前提条件 - TP-Link製ブロードバンドルーター([[0]]シリーズなど) - PPPoE接続用のアカウント情報(ユーザー名/パスワード) - 管理者権限でルーターにアクセス可能 ## 設定手順 1. **ルーター管理画面にログイン** - ブラウザで[[1]]または[[2]]にアクセス - 管理者ユーザー名とパスワードを入力 2. **DHCPサーバーを無効化** - 「ネットワーク」>「LAN」設定に移動 - 「DHCPサーバー」セクションで「無効」を選択 - 変更を保存 3. **PPPoE接続を設定** - 「ネットワーク」>「WAN」設定に移動 - 接続タイプで「PPPoE」を選択 - ISPから提供されたユーザー名とパスワードを入力 - 接続モードを「自動接続」に設定 4. **設定を適用** - すべての変更を保存 - ルーターを再起動 ## 注意事項 - DHCPを無効にすると、接続する端末で手動IPアドレス設定が必要になります - 静的IPアドレスはルーターのLANサブネット([[3]]など)と一致させる必要があります - 設定後は有線/無線接続の端末からインターネット接続を確認してください ## トラブルシューティング - 接続できない場合: 1. PPPoE認証情報を再確認 2. ルーターのファームウェアを最新版に更新[[4]] 3. ISPに接続障害がないか確認 この設定により、DHCPサービスを停止した状態でもPPPoE経由で安定したインターネット接続が可能になります。

TP-Linkの新しいルーターにはいくつか奇妙な制限があります。例えば、ルーターのDHCPサービスを無効にすると、すべてのポートがLanポートに変わってしまいます。

# Vueのリアクティブ原理と実装 Vue.jsの核心的な特徴の1つは、その**リアクティブシステム**です。このシステムにより、データが変更されると自動的にビューが更新されます。この記事では、Vueのリアクティブシステムがどのように動作するのか、その背後にある原理と簡易実装を解説します。 ## リアクティブシステムの基本概念 Vueのリアクティブシステムは主に以下の要素で構成されています： 1. **データオブザーバー** - データオブジェクトのプロパティを監視 2. **依存関係収集** - どのコンポーネントがどのデータに依存しているかを追跡 3. **ウォッチャー** - データ変更を監視し、対応する更新をトリガー ## 実装の核心：Object.defineProperty Vue 2.xでは、`Object.defineProperty`を使用してデータのgetter/setterを定義することでリアクティブを実現しています。以下はその簡易実装例です： ```javascript function defineReactive(obj, key, val) { // 依存関係を管理するDepクラス const dep = new Dep() Object.defineProperty(obj, key, { get() { // 依存関係を収集 if (Dep.target) { dep.depend() } return val }, set(newVal) { if (newVal === val) return val = newVal // 変更を通知 dep.notify() } }) } ``` ## 依存関係管理システム 依存関係を管理するための`Dep`クラスと`Watcher`クラスの簡易実装： ```javascript class Dep { constructor() { this.subscribers = [] } depend() { if (Dep.target && !this.subscribers.includes(Dep.target)) { this.subscribers.push(Dep.target) } } notify() { this.subscribers.forEach(sub => sub.update()) } } Dep.target = null class Watcher { constructor(vm, expOrFn, cb) { this.vm = vm this.getter = parsePath(expOrFn) this.cb = cb this.value = this.get() } get() { Dep.target = this const value = this.getter.call(this.vm, this.vm) Dep.target = null return value } update() { const oldValue = this.value this.value = this.get() this.cb.call(this.vm, this.value, oldValue) } } ``` ## Vue 3の進化：Proxyベースのリアクティブ Vue 3では、`Proxy`オブジェクトを使用してリアクティブシステムを再実装しています。これにより、以下の改善がもたらされました： - 配列の変更検出がより信頼性向上 - 新しいプロパティの追加も自動的に追跡可能 - パフォーマンスの向上 ```javascript function reactive(obj) { return new Proxy(obj, { get(target, key) { track(target, key) return Reflect.get(target, key) }, set(target, key, value) { Reflect.set(target, key, value) trigger(target, key) } }) } ``` ## まとめ Vueのリアクティブシステムは、データバインディングと自動更新の基盤です。その核心は： 1. データアクセスをインターセプトして依存関係を追跡 2. データ変更時に依存するウォッチャーに通知 3. 仮想DOMを効率的に更新 この原理を理解することで、Vueアプリケーションのデバッグやパフォーマンス最適化がより容易になります。

How to Enable Dial-Up Internet After Disabling DHCP on TP-Link When configuring a TP-Link router, you may need to disable its DHCP server to avoid conflicts in a network with multiple routers. However, this can prevent the router from automatically obtaining an IP address for internet access. Here's how to manually set up dial-up internet after disabling DHCP: ### Steps to Configure 1. **Access the Router's Admin Page** - Connect to the router via Wi-Fi or Ethernet. - Open a browser and enter the router's management address (e.g., `192.168.1.1`). - Log in with your admin credentials. 2. **Disable the DHCP Server** - Navigate to **Network > DHCP Settings**. - Set **DHCP Server** to **Disabled** and save the changes. 3. **Configure WAN Connection Manually** - Go to **Network > WAN Settings**. - Select **PPPoE** (or your ISP's required connection type). - Enter your ISP-provided username and password. - Set the connection mode to **Connect Automatically** or **Always On**. - Save the settings. 4. **Restart the Router** - Reboot the router to apply the changes. 5. **Verify Internet Access** - After restarting, check if the router successfully connects to the internet. ### Notes - If your network uses static IPs, configure the router's LAN IP to avoid conflicts. - Ensure the router's WAN port is connected to the modem or upstream network. By following these steps, your TP-Link router should maintain internet access even with DHCP disabled.

TP-Link's new routers have some peculiar limitations. For instance, if you disable the router's DHCP service, all ports will automatically switch to LAN ports.

# The Craft of Markdown and GitHub Markdown Markdown is a lightweight markup language widely used for formatting text. Its simplicity and readability make it popular for writing documentation, notes, and even blog posts. GitHub Markdown extends the basic syntax with additional features tailored for software development workflows. ## Key Features of Markdown - **Headers**: Use `#` for different levels of headings. - **Emphasis**: `*italic*` or `_italic_`, `**bold**` or `__bold__`. - **Lists**: - Unordered lists with `-`, `*`, or `+`. - Ordered lists with numbers. - **Links and Images**: - `[text](URL)` for links. - `![alt text](image URL)` for images. - **Code**: - Inline code with backticks: `` `code` ``. - Code blocks with triple backticks or indentation. ## GitHub-Flavored Markdown (GFM) GitHub enhances standard Markdown with features like: - **Task Lists**: `- [x] Completed task` / `- [ ] Pending task`. - **Tables**: ```markdown | Header 1 | Header 2 | |----------|----------| | Cell 1 | Cell 2 | ``` - **Strikethrough**: `~~strikethrough text~~`. - **Mentions & Issue References**: `@username`, `#issue-number`. - **Syntax Highlighting**: Specify language after triple backticks (e.g., ```python). ## Why Use Markdown? - **Portability**: Plain text works everywhere. - **Version Control Friendly**: Ideal for Git and GitHub. - **Efficiency**: Focus on content, not formatting.
